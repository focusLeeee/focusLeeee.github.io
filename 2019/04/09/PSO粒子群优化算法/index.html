PSO粒子群优化算法

发表于 2019-04-09 | 更新于 2019-04-11 | 评论数：

粒子群优化算法是一种模拟鸟群社会行为的群体搜索算法。粒子群的概念的最初意图是形象地模拟鸟群的优雅而不可预测的行为，目的是发现统御鸟群同步飞行的模式，以及在最优形式重组时突然改变方向的模式。

  • PSO算法对两个简单的行为建模
    • 每个个体向它邻近最好的个体移动；
    • 每个个体回退到该个体自己曾经到过的最好区域。
      因此出现的集体行为是所有的个体向环境中对所有个体都是最好的位置收敛。
  • 在PSO中，个体——粒子，在超维搜索空间中飞行。
    • 在搜索空间内粒子位置的改变是基于个体模仿其他粒子的成功的社会心里倾向；
    • 群内粒子的改变因此受其邻居的经验或知识影响。一个粒子的搜索行为因此受群中其他粒子的影响（因此PSO是一种共生协同算法）;
    • 对此社会行为的建模就是搜索过程中粒子随机地返回之前搜索过程中发现的搜索空间中的成功区域。

1. 基本粒子群优化

  • 粒子群中的个体有一个基本的行为:效仿领域个体的成功和他们自己的成功——从这个基本行为显现出来的集体行为就是在一个高维搜索空间中发现最优区域。
  • 一个PSO算法维护一群粒子，其中每个粒子表示一个潜在的解。
  • 粒子在一个多维搜索空间中飞行，而每个粒子的位置根据它自身的经验及其邻居的经验来调整。令$x_i(t)$表示粒子$i$在时刻$t$时在搜索空间中的位置，$t$离散的时间步。粒子位置通过对当前的位置增加加速度$v_i(t)$来改变，即：式中$x_i(0)\sim U(x_min, x_max)$
  • 驱动优化过程的是速度向量，速度向量反应了粒子的经验知识和粒子的领域的社会交换信息。
    • 粒子的经验知识即认知部分，它与粒子离它从开始找到过的最佳位置（个体最佳位置）的距离成正比。
    • 社会交换信息叫做速度方程的社会部分。
  • gbest PSO和lbest PSO是领域大小不同的两个PSO算法。

  • 使用粒子索引来选取邻域的原因
    • 由于不需要粒子的控件顺序故计算代价小。对于利用粒子之间的距离来形成邻域的方法，需要计算所有粒子对之间的欧式距离，其时间代价为$O(n_s^2)$
    • 不管粒子在搜索空间的审美位置，它有助于将了解的信息传播到所有的粒子。
  • 还需要注意邻域的重叠问题，一个粒子是多个邻域中的一员。
    • 这种邻域的关联性也有助于邻域之间的信息共享，并保证群体向一个单独的点收敛，即全局最佳粒子
    • gbest PSO是lbest PSO的特例，其中$n_{\mathcal{N_i}} = n_s$

1.3 比较gbest PSO和lbest PSO

  • 相同点
    • 两种都向全局最佳粒子靠近（lbest PSO中邻域重叠）
  • 不同点
    • gbest PSO有更大的粒子互联度，因此比lbest PSO收敛速度快。但代价是比lbest PSO有更差的多样性。
    • lbest PSO不容易先于局部极值，一般来说（取决于问题），lbest PSO中使用的环型拓扑的邻域结构能改善性能

1.4 速度成分

公式(2),(5)的速度计算都由三部分组成：

  • 先前的速度$v_i(t)$,作为之前的飞行方向的记忆，即刚刚过去的移动。这一记忆项可以被看做是一个惯性，防止粒子急剧改变方向，并偏离当前方向，也叫做惯性分量。
  • 认知分量$c_1r_1(y_i-x_i)$，计算量粒子$i$与过去性能相关的性能。在某种程度上，认知分量模拟了该粒子最好位置的记忆。该项的作用是粒子可以被拉往自己的最佳位置，表示粒子回到过去最为满足他们的位置或场景，称认知分量为粒子的“念旧”。
  • 社会分量,在gbest PSO中是$c_2r_2(\hat{y} - x_i)$，在lbest PSO中是$c_2r_2(\hat{y}-x_i)$。它计算了粒子$i$相对于一组粒子（或邻域）的性能。因此，从概念上说，社会分量模拟了个体试图达到的群体中的规范和标准。社会分量的作用是每个粒子被拉向该粒子的邻域所找到的最好的解。

认知分量和社会分量的贡献分别由随机数$c_1r_1$和$c_2r_2$来控制

1.6 算法有关方面

对前文两个算法，还需讨论粒子初始化、终止条件、定义迭代和函数评估

  • 最优化过程是迭代的——算法执行重复的迭代，直到满足某个终止条件
  • 一次迭代由$repeat$和$until$循环内的所有步骤构成：决定个体最佳位置和全局最佳位置、调整每个粒子的速度、执行函数评估
  • 一个函数评估（FE）是指适应度函数的计算，适应度函数代表了具体的优化问题（对基本PSO，每一次执行总共$n_s$次函数评估，$n_s$是群中粒子的个数）
  • PSO算法第一步：初始化群和控制参数。
    • 在基本PSO情景中，加速常数$c_1$和$c_2$、初始速度、粒子位置和个体最佳位置都需要被赋初值，lbest PSO还需指定邻域粒子个数，
    • 粒子位置：通常初始化为在搜索空间均匀分布（PSO效率受初始群的多样性影响，即覆盖了多少搜索空间，且粒子在搜索空间内时如何分布的。若搜索空间区域未被初始群覆盖，而极值正好在未覆盖区域，则PSO很难找到这个极值——除非粒子的惯性将粒子带到该未覆盖区域，并且最后找到一个新的个体最佳值，或称为全局最佳值。）
    • 假设极值在两个向量$x_{min}$和$x_{max}$所定义的区域内，则这两个向量分布表示在每个纬度上的最小值和最大值，那么粒子位置的一个有效初始方法为：其中$r_j \sim U(0, 1)$,初始化速度可初始化为0：$v_i(0) = 0$
      1. 初始化也可为随机值，但是一方面没有必要，另一方面大的初始速度会有大的初始惯性，因此导致大的初始位置更新，容易引起粒子离开搜索空间的边界，可能使群需要经过更多代才能收敛到某个解

      3. 初始化粒子位置还有sobol序列，faure序列，非线性单纯型等方法。
      4. 均匀分布的粒子并非最好。均匀初始化会导致粒子在即将优化的函数的最优值附近对称分布，因此建议在不包含极值的区域初始化，以验证算法定位初始化区域之外的解的能力。
  • 终止条件——用于终止迭代搜索过程的准则
    • 需考虑两个方面
      • 终止条件不应导致PSO早熟，否则会得到次优解
      • 终止条件需要放置对适应度函数过度取样。如果一个终止条件需要频繁的适应度函数计算，搜索过程的计算复杂性会急剧增加。
    • 已用过的终止条件
      • 当超过迭代最大次数或函数评估最大次数时，终止：通常与收敛准则共同使用；适用于在研究目标为评估限制时间周期内获得的最好解时
      • 当找到一个可接受的解时终止：假设$x^$表示目标函数$f$的最优点。那么只要找到一个粒子$x_i$使得$f(x_i) \leq |f(x^ - \varepsilon)|$,则该准则终止搜索过程。如果$\varepsilon$太大，则终止于一个坏的次优解；若太小，搜索可能无法终止。
      • 一定次数的迭代内没有改进时终止：粒子位置的平均改变很小；一定迭代次数后平均粒子速度为0；一定迭代次数后没有显著改进。引入了两个参数值：(1)监测性能的迭代(函数评估)窗口(2)指示性能无法接受的阈值
      • 当归一化群半径接近于0时终止：归一化群半径可计算为：$diameter(S)$是初始化群的直径，最大半径$R_{max}$为：$R_{max} = ||x_m - \hat{y}||, m = 1, …, n_s$,其中：$||x_m - \hat{y}|| \ge ||x_i - \hat{y}||, \forall i = 1, …, n_s$.如果归一化群半径接近于0，群不太可能继续改进，除非全局最优还在移动。$||*||$是某种合适的距离尺度，如欧式距离。
        算法在$R_{norm} < \varepsilon$时终止，如果$\varepsilon$太大，则终止于一个坏的次优解；若太小，搜索可能要花费过多次迭代才能使粒子形成一个紧凑群，紧密地围绕在全局最优位置周围。
      • 当目标函数坡度近似为0时终止。上面的终止条件考虑了搜索空间粒子的相对位置，但没有考虑关于目标函数坡度的信息。为目标函数的变化率作为终止的根据，考虑比例：如果对一系列连续迭代，有$f’(t) < \varepsilon$,则认为群已收敛。（此方法比上述方法好，因为它实际决定了群是否仍然在利用搜索空间相关的信息进行优化；但问题是可能会陷入局部极值，因此可以联合半径及聚类方法使用，来测试是否所有粒子在搜索空间停止前收敛到了同一个点）
  • 该算法的结果是一个单独的类C

2. 社会网络结构

  • 驱动PSO的关键特征是社会交互作用。
    • 群内的粒子彼此相互学习，并且根据获得的知识基础，移动以变得更加近似它们的更好邻居。
    • PSO的社会结构决定于重叠邻域的形成。其中邻域内的粒子相互影响。
    • 一个有机体最有可能被其邻域中的其他人影响，并且更加成功的有机体将对邻域中相对不成功的成员施加更大的影响。
  • PSO内，相同邻域内的粒子通过交换邻域中每个粒子的成功信息来彼此通信。然后所有粒子移向被认为是更好的位置。
    • PSO的性能强烈依赖于社会网络结构。
    • 通过社会网络流动的信息流依赖于：
      • 网络结点(成员)之间的连接度
      • 聚类数(当一个节点的邻居同时是另外一个节点的邻居时发生聚类)
      • 节点之间的平均最短距离
    • 高度连接的网络结构
      • 更快收敛
      • 更容易陷入局部极值(在搜索空间中的覆盖范围比弱连接网络更小)
    • 稀疏连接网络
      • 搜索空间没有充分覆盖而无法获得最佳的可能解
      • 收敛缓慢
  • 部分社会网络结构
    • 星型社会结构
      • 所有粒子相互连接，如16.4(a)
      • 每个粒子都被吸引向整个群找到最佳粒子
      • 收敛更快，可能陷入局部极值。gbest
    • 环型社会结构
      • 16.4(b)，每个粒子跟他的$n_N$个直接邻域通信
      • 信息在社会网络中以更慢的速度流动，收敛也更慢
      • 但与星型相比，更大部分的搜索空间被覆盖.lbest
    • 轮型社会结构
      • 16.4(c)邻域中的个体彼此孤立，一个粒子充当聚焦点，所有信息通过这个聚焦粒子来通信
      • 聚焦离子比较邻域中所有粒子的性能，根据最好的邻居调整自己的位置
      • 如果聚焦点的新的位置代表了更好的解，这个改进传递给邻域中的所有成员。
      • 降低了群中好的解的传播速度
    • 金字塔型社会结构
      • 16.4(d)三维有线连接框架
    • 四聚类型社会结构
      • 16.4(e)在这个社会网络中，4个聚类由聚类之间的两个连接形成。一个聚类内的粒子跟5个邻居相连。
    • 冯诺依曼型社会结构
      • 16.4(f)其中粒子连接成一个栅格结构。
      • 在一些实验研究中，冯诺依曼社会结构被证明为在解决大量问题中都比其他社会结构好
  • 不存在对所有问题都适用的最好拓扑
    • 依赖于粒子之间的互连接度
    • 全连接的结构对单模问题有最好的性能
    • 非全连接网络对多模问题有更好性能
  • 邻域通常根据粒子的索引决定
    • 也有根据欧式距离决定

3. 基本变种

  • 基本PSO被成功应用到了，标准函数优化、解决排列问题，训练多层神经网络等问题上。
  • 为了改进收敛速度和PSO找到解的质量，提出如下修改
    • 引入惯性权重
    • 引入速度约束
    • 引入决定个体最佳和全局最佳的不同方法
    • 引入不同速度模型

3.1 velocity clamping

  • 决定一个优化算法的效率和精确率的一个重要方面就是探索和挖掘的平衡(PSO内，这些目标通过速度更新方程来处理)
    • 探索是一个搜索算法探索搜索空间不同区域以定位一个好的优化解的能力
    • 挖掘是专注于搜索某个区域以提高候选解的精确率的能力。
  • 标准PSO的早期应用中，速度很快变为大的值，特别是原理全局最佳和个体最佳位置的粒子。因此粒子位置更新很大，导致粒子离开搜索空间的边界——粒子偏离。
